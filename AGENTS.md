# 에이전트 안내

## 프로젝트 개요
- 목적: 대시보드 콘솔과 API 서버를 배포하기 위한 설치/설정 자동화 스크립트 묶음.
- 핵심 동작: 최신 웹 릴리즈 다운로드, API 도커 이미지 가져오기, 환경/도커 설정 파일 생성.
- 실행 대상: 리눅스 루트 권한 필요. Docker 및 Docker Compose v2 필요.

## 디렉터리 구조
- `setup.sh`: 전체 설치 흐름을 실행하는 단일 스크립트.
- `output/`: 실행 결과물(생성 파일). **수정 금지**.
- `templates/`: 템플릿 보관용 디렉터리(현재 스크립트는 heredoc로 생성).

## 실행 흐름 요약
1) `setup.sh` 실행  
2) 사전 점검  
3) 최신 릴리즈 다운로드 및 이미지 가져오기  
4) `output/docker`에 Caddy/Docker Compose/환경 파일 생성  

## 변경 시 주의사항
- `output/`은 생성 결과물이며 소스가 아니다.
- 스크립트는 `set -euo pipefail`로 동작하므로 변수 누락/명령 실패에 민감하다.
- 기본 입력값, 환경변수, 출력 경로 변경 시 **setup.sh → 각 스크립트**의 흐름을 함께 검토한다.

## 커밋 및 PR 규칙
- 커밋 메시지 헤더는 영문 타입 + 선택 스코프를 PascalCase로 표기하고, 본문은 한글로 작성한다.  
  포맷: `<Type(Scope)>:한글 요약`  
  예) `Doc(timetable):시간표 API 문서 동기화`, `Feat(classroom):강의실 목록 위젯 추가`
- 타입 예시: `Feat`, `Fix`, `Doc`, `Refactor`, `Chore`, `Test`.
- 스코프는 snake_case 디렉터리나 기능명을 소문자로 표기한다.
- 커밋 본문에는 한 줄당 하나씩 핵심 변경사항을 남긴다.

## 커뮤니케이션 원칙
- 사용자 지원 시 **한국어로 응답**한다.
- 누락된 정보가 있으면 **먼저 질문**하여 요구사항을 명확히 한다.

## 문서 작성 규칙
